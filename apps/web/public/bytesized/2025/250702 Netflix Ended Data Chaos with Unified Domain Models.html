TL;DR

On June 12, 2025, Netflix unveiled UDA (Unified Data Architecture) – a foundational system that models business entities like ‘movie’ or ‘actor’ once and projects them consistently across APIs, databases, and pipelines. By combining knowledge graphs, automated schema generation, and domain-driven design, Netflix tackles the problem of fragmented data models that plague large-scale distributed systems.

Why does this matter?

For senior engineers managing data platforms, UDA solves a problem that emerges as organizations scale:

At Netflix, core concepts like ‘movie’ or ‘actor’ were redefined independently in each system, the GraphQL gateway powering internal tools, the media asset management platform storing content, the encoding pipeline transforming assets, and countless microservices. Each team modelled the same entities differently. The result? Inconsistent definitions, terminology drift, data quality issues, broken references, and brittle integrations that require heroic manual efforts to patch.

For any company with rapidly growing microservices and domain teams, these challenges are inevitable. UDA is Netflix’s approach to future-proofing data interoperability at scale.

What is UDA?

Netflix describes UDA as a “foundation for connected data”. Conceptually, it is a knowledge graph–driven system where:

  1. Domain models are defined once – formal conceptualizations of business entities like movies, actors, or games.

  2. These models are projected automatically into concrete schemas – GraphQL, Avro, SQL, Java – preserving their semantics everywhere they are used.

  3. Mappings connect models to real data containers – like GraphQL resolvers, Data Mesh sources, or Iceberg tables, enabling systems to find and use data consistently.

  4. Data moves faithfully between systems, with meaning preserved, without brittle manual ETL transformations.

  5. Everything is organized as a knowledge graph, enabling search, introspection, and programmatic graph traversal to uncover relationships across systems.

Upper: The Metamodel Engine

At the heart of UDA lies Upper, Netflix’s metamodeling framework. It defines the language for domain modeling:

Upper itself is a bootstrapping metamodel – it models itself, defines what a domain model is, and validates its own definitions. This self-referencing design allows Netflix to generate:

  • Type-safe Java APIs for domain models.

  • GraphQL schemas federated into their Enterprise Gateway.

  • Avro schemas for Data Mesh ingestion.

Because all models are conservative extensions of Upper, new domains integrate seamlessly into Netflix’s runtime and developer tooling.

How UDA Changes Data Engineering at Netflix

Netflix built UDA to solve tangible problems faced by teams building and operating large data pipelines:

Previously, when a business user wanted to generate an operational report, they had to know which tables to query, how to join them, and what terminology each system used. Now, with Sphere, Netflix’s operational reporting tool powered by UDA, users can search for familiar business concepts like ‘actors’ or ‘movies’. Sphere traverses UDA’s knowledge graph to resolve these to the correct data containers and generates SQL queries automatically, ensuring semantic accuracy and eliminating reliance on data engineers for every new report.

In another use case, Primary Data Management (PDM) uses UDA to manage controlled vocabularies and taxonomies. For example, One Piece character hierarchies are defined as domain models. UDA then projects these into GraphQL APIs and Avro schemas, automatically provisioning the APIs and data products needed downstream, with guaranteed consistency across teams.

Why is this a big deal for system and data architects?

  1. Eliminates schema drift and integration debt. UDA’s “model once, project everywhere” approach ends the nightmare of reconciling duplicated models across systems.

  2. Automates semantic data integration. By encoding meaning explicitly via mappings, UDA enables intent-based pipelines where data moves between systems with fidelity, without fragile manual transformations.

  3. Decouples business logic from implementation details. Business entities are defined at the conceptual level, and UDA handles generating the necessary schemas, mappings, and pipelines to materialize them across heterogeneous systems.

  4. Improves developer velocity and operational consistency. New features or schema changes are defined once and propagate across APIs, data lakes, and services without conflicting definitions or last-minute integration bugs.

  5. Future-proofs large-scale distributed architectures. As Netflix expands into games, live events, and ads, UDA ensures that new domains integrate cleanly without reintroducing legacy model fragmentation.

The Broader Lesson

For any organization approaching Netflix-scale complexity, UDA offers a blueprint for integrating domain-driven design, knowledge graphs, and automated schema generation to build scalable, consistent, and future-proof data architectures.

“Distributed systems are resilient until fragmented models and inconsistent semantics break them from within. UDA shows that uptime is also about everyone meaning the same thing when they say ‘movie’ or ‘actor’.”
